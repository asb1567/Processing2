# Processing2

### 도형 그리기

- **트라이앵글(Triangle)**

       3개의 점을 연결하여 삼각형을 그리는 함수이다

       매개변수로는 각 점의 x, y 좌표가 필요하다

- **쿼드(Quad)**

       4개의 점을 연결하여 사각형을 그리는 함수이다

### 도형 그리기 명령어 비교

- **엘립스(Ellipse) vs 써클(Circle)**

       엘립스 : 타원을 그리는 명령어, 매개변수로는 중심점의 x 좌표, y 좌표, 너비, 높이를 입력한다

       써클 : 원을 그리는 명령어, 중심점의 x 좌표, y 좌표, 그리고 반지름을 입력한다

- **스퀘어(Square) vs 렉트(Rect)**

       스퀘어 : 정사각형을 그리는 명령어, 중심점의 x 좌표, y 좌표, 그리고 한 변의 길이를 입력한다

       렉트 : 사각형을 그리는 명령어, x 좌표, y 좌표, 너비, 높이를 입력한다

### 렉트**(Rect)**와 엘립스**(Ellipse)**

- **렉트(Rect)**

       사각형을 그리는 명령어이다**

       매개변수로는 코너(Corner), 코너스(Corners), 레이어스(Radius), 센터(Center)가 있다

- **엘립스(Ellipse)**

       타원을 그리는 명령어이다

       매개변수로는 코너(Corner), 코너스(Corners), 레이어스(Radius), 센터(Center)가 있다

### 비긴 쉐이프**(Begin Shape)**와 앤드 쉐이프**(End Shape)**

- **비긴 쉐이프(Begin Shape)**

       그림을 그리기 시작할 때 사용한다

- **엔드 쉐이프(End Shape)**

       그림 그리기를 종료할 때 사용한다

### 도형 및 색상 설정을 설명하는 로고 디자인 과정

- **‘앤드 쉐이프(End Shape)’ 명령어를 사용하여 다각형을 그릴 수 있다**

- **‘비긴 쉐이프(Begin Shape)’ 명령어를 사용하기 전에 no를 넣으면 색이 자동으로 칠해지지 않다**

- **눈을 그릴 때는 선의 두께를 이용하여 점을 찍고 스트로크 에이트를 조절하여 눈을 그린다**

- **'백그라운드' 명령어를 사용하여 배경색을 깨끗하게 지울 수 있다**

- **색상 선택 방법은 HSB를 사용할 수 있고 H 값은 색상을, S 값은 채도를, B 값은 밝기를 나타낸다**

- **HSB에서 왼쪽은 0, 오른쪽은 99까지의 범위를 가지며, 위로 갈수록 값을 증가한다**

### 색상 선택과 코딩 프로세싱에 관한 설명

- **HSB 색상 선택 방법은 색상, 채도, 명도를 조절하여 색상을 선택한다**

- ‘**컬러 모드' 명령어를 사용하여 RGB 또는 HSB 중 하나를 선택하여 색상을 선택한다**

- **RGB로 색을 선택한 경우, HSB로 변환하면 채도와 명도를 조절하여 다른 색상을 선택할 수 있다**

- **색상 선택 창에서 H 값은 360도, 채도와 명도는 0에서 99 사이의 값을 가진다**

- **코드를 사용하여 도형을 그릴 때는 '마우스 x'와 '마우스 y'를 사용하여 마우스의 위치에 따라 도형**

  **을 그릴 수 있다**

### 인터랙티브 소프트웨어 개발 과정

- **프로그램은 마우스의 움직임에 따라 원을 그리는 인터랙티브 소프트웨어이다**

- **'높이' 명령어를 사용하여 외곽선을 지우고, '은' 명령어를 사용하여 원을 반투명하게 그린다**

- **화면을 반투명한 검정색으로 지우고, 흰색으로 반투명하게 원을 그린 후, 반투명하게 다시 지우는**

  **과정을 반복하여 잔상을 만든다**

- **프로그램을 수정하고 가독성을 높이기 위해 들여쓰기를 사용한다**

- **원이 회색으로 보이는 이유는 반투명하게 지워진 영역 때문이므로, 투명도를 없애서 완전한 흰색**

  **으로 바꿔준다**

### 프로세싱으로 실시간 그림 그리기

- **현재의 프로그램 순서**
    1. **흰색으로 그리고, 지우면서, 그리고, 60초 대기**
    2. **흰색으로 그리고, 치우고, 다시 60초 대기**
    3. **이 과정을 반복**

- **원하는 동작**
    
    **먼저, 이전에 그려진 그림을 반투명하게 지운 후 흰색으로 그린다**
    
    **마우스의 이동에 따라 선이 쭉 연결되는 프로그램을 만들고자 한다**
    
- **실행되는 로직**
    
    **윈도우를 생성하고, 흰색으로 지우고 그림을 그리며, 60프레임/초로 동작한다**
    
    **마우스의 위치를 기반으로 이전 위치부터 현재 위치까지 선을 연결하여 그린다**
    

### 프로세싱을 활용한 마우스 위치에 따라 런버블 로고 그리기

```c
// 이 코드는 러버블랙 로고를 그리고, 지우고 선 두께를 정하며, 러버블 로고를 입부분과 눈을 그리는 것을 포함합니다.

// 백그라운드 색상을 설정합니다.

// 라인의 두께를 설정합니다.

// 로고를 그립니다.
// 입 부분

// 눈 부분

line(mouseX + 20, mouseY - 30, mouseX + 20, mouseY + 30);

// 손 두껍게 설정
strokeWeight(10);
// 러버블 로고를 그립니다.

// 마우스 위치에 따라 러버블 로고가 이동하도록 설정합니다.

// 눈 부분
line(mouseX - 20, mouseY - 30, mouseX - 20, mouseY + 30);
line(mouseX + 20, mouseY - 30, mouseX + 20, mouseY + 30);

// 주석을 추가하여 코드를 설명합니다.
// '/'로 시작해서 '/'로 끝나는 주석은 프로세싱에서 무시됩니다.
// 이렇게 설명을 추가하면 코드를 이해하기 쉬워집니다.
```

### 마우스 위치에 따라 동적으로 점 이동 시키기

- **코드 개요**

      **마우스 위치에 따라 다양한 위치에 점을 그리고 이동 시키는 역할을 한다**

- **주요 기능**
    
    **점들은 마우스 위치를 기준으로 상대적인 위치에 따라 이동한다**
    
    **각 점은 마우스의 위치와 일정한 거리만큼 떨어진 위치에 그려진다**
    
    **마우스의 위치와 상대적인 거리는 코드를 통해 계산된다**
    

- **코드 설명**
    
    **마우스의 위치를 기준으로 점들의 상대적인 위치를 계산하는 과정이 주석으로 설명되어 있다**
    
    **마우스 위치와의 상대적인 거리를 계산하기 위해 각 점의 초기 위치에서 일정한 값을 더하거나 빼**
    
    **는 방식이 사용됐다**
    

- **동작 확인**
    
    **코드를 실행하면 마우스를 움직일 때마다 점들이 마우스의 위치를 기준으로 이동한다**
    
    **마우스를 움직이는 동안 각 점의 위치가 동적으로 변화하는 것을 확인할 수 있다**
    

### 코드 상으로 더하고 빼고 곱하고 나누는 방법 설명

- **프로그램 코드에서는 사칙연산을 활용하여 마우스 위치에 따라 원하는 위치를 계산할 수 있다**

- **이를 통해 그림이 상대적으로 이동하도록 만들 수 있다**

- **곱하기와 나누기가 먼저 실행되고, 그 다음에 더하기와 빼기가 실행된다**

- **괄호를 사용하여 연산 순서를 조절할 수도 있다**

- **괄호를 사용할 때는 시작하는 괄호와 대응하는 닫히는 괄호를 반드시 쌍으로 사용해야 한다**

### 4층 연산을 이용한 애니메이션 만들기

1. **배경을 검정색으로 설정하고, 화면 좌측 상단에 원을 그린다**
2. **원을 오른쪽으로 이동 시키는 애니메이션을 만들기 위해 변수 사용이 필요하다**
3. **변수 이름은 사용자가 선택하되, 해당 값이 마우스의 x좌표와 같이 변화하도록 설정한다**

### 변수와 정수값 컴퓨터 메모리에서 저장하는 방법

1. **x 포지션 변수를 생성하여 정수를 저장한다**
2. **프로그램에서 x 포지션 변수를 사용하여 값을 변경하거나 활용한다**
3. **변수를 선언할 때 int 키워드를 사용하여 해당 변수가 정수를 저장한다는 것을 명시한다**
4. **변수를 처음에는 0으로 초기화하고, 프로그램 실행 중에 필요에 따라 값을 변경할 수 있다**
5. **변수를 사용하여 화면의 픽셀 위치를 제어하고, 위치를 변경함으로써 애니메이션 효과를 만들 수** 

   **있다**

### 프로세싱 변수 : int부터 float까지

- **변수를 선언할 때 int 키워드를 사용하여 정수를 저장한다**

- **변수를 사용하여 화면의 픽셀 위치를 제어하고, 위치를 변경하여 애니메이션 효과를 만든다**

- **int 변수는 정수를, float 변수는 소수를 저장한다**

### 프로그래밍 변수 이름 설정 규칙과 시스템 변수 활용

- **불리언 변수는 컴퓨터에서 가장 기본적이고 참과 거짓을 저장한다**

- **자바의 변수는 -128부터 127까지의 값을 저장할 수 있다**

- **캐릭터 변수는 문자 하나를 저장할 수 있다**

- **더블 변수는 플로트 변수와 유사하다**

- **롱 변수는 더 넓은 범위의 정수를 저장할 수 있다**

### 마우스를 따라 색상 변화 만들기

1. **프레임 카운트 시스템 변수를 활용하여 색상을 조절한다**
2. **프레임 카운트는 프로그램이 실행될 때마다 1씩 증가한다**
3. **4의 배수마다 빨간색에서 시작하여 빨, 주, 노, 초, 파, 남, 보, 백, 상 순으로 색이 변화한다**
4. **이후 다시 처음 색상으로 돌아오며, 이러한 변화를 반복한다**
5. **프레임 카운트를 256으로 나눈 나머지를 활용하여 0부터 255까지의 범위를 반복하도록 한다**
6. **결과적으로 색상이 변하면서 밝기가 깜빡거리는 효과를 얻을 수 있다**

### 화면 중간 이동 후, 랜덤 밝기 색 지정하기

- **화면 중앙에 요소를 배치할 때는 시스템 변수를 활용하여 윈도우 크기에 관계없이 자동으로 중앙**

  **에 위치하도록 코드를 작성한다**

- **색상을 지정할 때는 색상 채도와 명도를 활용하며, 명도 값을 랜덤으로 변경하여 다양한 효과를**

  **표현할 수 있다**

- **랜덤 함수를 사용할 때는 0부터 255 사이의 값을 생성하여 밝기 값을 조절한다**

- **랜덤 함수는 다양한 상황에서 유용하게 활용될 수 있다**

### 조건문을 활용한 마우스 위치에 따른 색상 변경

- **조건문 소개**
    
    **조건문은 특정한 상황에서만 실행되는 코드를 작성할 때 사용된다**
    
    **조건문은 불리언(boolean) 변수와 밀접한 연관이 있다**
    
    **프로세싱에서 참은 true, 거짓은 false로 표현된다**
    
- **기본 문법**
    
    **if (조건) { 실행 코드 }: 조건이 참일 경우에만 실행된다**
    
    **else { 실행 코드 }: 조건이 거짓일 경우 실행된다**
    
- **코드 작성 시 주의할 점**
    
    **if문 안의 코드가 참이 아닌 경우에는 해당 코드가 실행되지 않는다**
    
    **if문 안의 조건이 참일 때만 실행되는 코드를 작성해야 한다**
    

### 조건문을 활용한 구조화된 코드 작성하기

- **중요한 부분 강조**
    
    **코드 정렬을 통해 코드의 구조를 명확히 확인할 수 있다**
    
    **코드의 일관된 들여쓰기는 가독성을 향상시킨다**
    
    **주석을 추가하여 코드의 각 부분이 어떤 역할을 하는지 설명해야 한다**
    
- **조건문의 올바른 사용**
    
    **if-else 구문을 사용하여 마우스 위치에 따라 배경 색을 변경하는 예시를 보여준다**
    
    **적절한 들여쓰기와 주석을 통해 코드를 명확하게 설명해야 한다**
    
- **코드 구조화**
    
    **코드를 여러 단계로 분할하여 가독성을 높인다**
    
- **추가적인 팁**
    
    **코드를 읽고 수정할 때는 항상 코드 정렬을 사용해야 한다**
    
    **들여쓰기와 주석을 통해 코드의 의도를 명확히 전달해야 한다**
    
    **여러 개의 조건문을 사용하여 복잡한 작업을 수행할 수 있다**
    

### 조건문을 활용한 코드의 가독성과 구조화

- **구조화된 코드**
    
    **코드를 구조화하여 가독성을 높인다**
    
    **정렬된 코드와 일관된 들여쓰기를 사용해야 한다**
    
- **주석을 활용한 설명**
    
    **각 코드 블록에 주석을 추가하여 코드의 의도를 명확히 전달해야 한다**
    
    **코드의 각 부분이 어떤 역할을 하는지 설명해야 한다**
    
- **조건문의 활용**
    
    **마우스 위치에 따라 배경 색을 변경하는 조건문을 예시로 제시한다**
    
    **조건문을 사용하여 다양한 상황에 대응한다**
    
- **코드의 분할**
    
    **코드를 여러 단계로 나누어 복잡성을 줄인다**
    
    **다섯 단계의 색상을 적용하는 예시를 통해 코드를 설명한다**
    
- **유용한 팁**
    
    **코드를 수정할 때는 항상 코드 정렬을 사용해야 한다**
    
    **코드의 가독성을 높이기 위해 들여쓰기와 주석을 적절히 활용해야 한다**
    

### 조건문과 비교 연산자의 활용

- **조건문의 다양한 활용**
    
    **if-else 구문을 사용하여 다양한 상황에 대응한다**

    **두 값이 같은지, 다른지, 작은지, 작거나 같은지, 큰지, 크거나 같은 지를 확인한다**
    
- **논리 연산자의 활용**

### 프로세싱을 사용한 원의 화면 내 이동 제어하기

- **목표**
    
    **원이 화면의 양쪽 벽에 부딪치면서 왔다 갔다 하도록 만든다**
    
    **원의 이동 방향을 바꾸는 조건을 설정하여 구현한다**
    
- **변수 설정**
    
    **스피드 변수를 만들어 원의 이동 속도를 조절한다**
    
- **원의 이동 제어**
    
    **x 포지션에 스피드를 더하여 원을 오른쪽으로 이동한다**
    
    **화면을 벗어나면 스피드 값을 음수로 변경하여 반대 방향으로 이동하도록 한다**
    
    **왼쪽 벽에 부딪치면 다시 양수로 변경하여 오른쪽으로 이동하도록 한다**
    
- **코드 작성 예시**
    
    **if 구문을 사용하여 화면 밖으로 나가는 경우와 왼쪽 벽에 부딪치는 경우를 처리한다**
    
    **원의 이동 방향을 반대로 바꾸는 방법을 설명한다**
    

### 프로세싱을 사용한 반복문의 이해와 활용

- **반복문의 중요성**
    
    **컴퓨터 프로그래밍에서 반복문은 가장 중요한 요소 중 하나이다**
    
    **반복적인 작업을 효율적으로 처리할 수 있도록 해준다**
    
- **반복문의 활용**
    
    **프로세싱을 사용하여 비주얼 작업을 할 때 반복문은 매우 유용하다**
    
    **많은 파티클을 생성하거나 특정 패턴을 만드는 작업에서 반복문이 사용된다**
    
- **반복문의 종류**
    
    **while 루프는 조건을 만족할 때까지 코드를 반복 실행한다**
    
    **이때 조건이 항상 참이면 무한 루프에 빠질 수 있으므로 주의해야 한다**
    
- **주의할 점**
    
    **무한 루프에 빠지는 것을 피하기 위해 조건을 제대로 설정해야 한다**
    
    **조건이 항상 참이 되는 경우를 피하여 코드가 영원히 실행되는 상황을 방지해야 한다**
    

### 반복문을 사용하여 점 찍기

```java
javaCopy code
void setup() {
  size(500, 500);
  background(255);
  strokeWeight(10);
  // p(250, 20); // 처음에 점 하나 찍기
  // p(270, 20); // 20픽셀 간격으로 점 찍기
  // p(290, 20);
  // ... 이런식으로 수작업으로 점을 찍을 수 있지만 번거롭습니다.

  int x = 250; // 초기 x 좌표
  int y = 20;  // 고정된 y 좌표
  int gap = 20; // 점 사이의 간격

  while (x <= 490) { // x 좌표가 490을 넘지 않을 때까지 반복
    p(x, y); // 점 찍기
    x += gap; // x 좌표를 간격만큼 증가
  }
}

void p(int x, int y) {
  point(x, y); // 점 찍기
}
```

- **반복문을 사용한 점 찍기 결과**

       x 좌표가 20픽셀 간격으로 증가하면서 오른쪽으로 이동하는 점이 찍힌다

### 변수를 사용한 반복문

```java
void setup() {
  size(500, 500);

  }

  point(x, y); // 점 찍기
}

```

- **변수를 사용한 반복문 결과**

        x 좌표가 20부터 시작하여 20씩 증가하면서 오른쪽으로 이동하는 점이 찍힌다

       간격을 10으로 변경하면 간격이 좁아지는 것을 확인할 수 있다

### for 반복문을 사용한 점 찍기

```java
javaCopy code
void setup() {
  size(500, 500);
  background(255);
  strokeWeight(10);

  for (float x = 20; x < 500; x += 20) { // x 좌표가 500 미만인 동안 반복
    point(x, 20); // 점 찍기
  }
}

```

- **for 반복문 결과**

       for 반복문을 사용하여 간결하고 가독성이 좋은 코드로 점을 찍었다

       코드의 의도가 명확하게 드러나며, 반복 횟수를 명시적으로 지정했다
